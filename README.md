# Telegram-бот для записи на визу VFS Global

Бот на **aiogram 3.x**, который собирает заявки на запись в визовый центр
VFS Global, хранит их в **PostgreSQL** и периодически проверяет свободные
слоты через браузерную автоматизацию (**Playwright**). При нахождении слота
бот пытается оформить запись и уведомляет пользователя.

> ⚠️ **Важно.** Браузерный слой VFS (`browser/client.py`) — это
> документированный **каркас со стабами**. Реальные селекторы, сценарий
> авторизации и обход анти-бот защиты намеренно не реализованы: вёрстка
> VFS закрыта, зависит от страны/центра и периодически меняется. Перед
> использованием изучите Условия использования VFS Global и применимое
> законодательство — автоматизация записи и обход капчи могут им
> противоречить.

## Возможности

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и список команд |
| `/add` | Создание заявки: диапазон дат + персональные данные через FSM |
| `/status` | Статусы всех заявок пользователя |
| `/edit` | Изменение диапазона дат или отдельного поля заявки |
| `/cancel` | Отмена активной заявки |
| `/help` | Краткая справка |

## Требования

- Python 3.11+
- PostgreSQL 13+

## Установка

```bash
# 1. Виртуальное окружение
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 2. Зависимости
pip install -r requirements.txt

# 3. Браузеры Playwright
playwright install chromium

# 4. Конфигурация
cp .env.example .env
# затем отредактируйте .env: BOT_TOKEN, DATABASE_URL и прочее
```

## Настройка `.env`

Ключевые переменные (полный список — в `.env.example`):

- `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather);
- `DATABASE_URL` — строка подключения PostgreSQL (драйвер `asyncpg`);
- `VFS_BASE_URL` — URL визового центра;
- `CHECK_INTERVAL_SECONDS` — интервал проверки слотов;
- `BROWSER_HEADLESS` — запуск браузера без окна.

## Запуск

```bash
python main.py
```

Таблицы БД создаются автоматически при первом запуске. Для продакшена
рекомендуется перейти на миграции **Alembic** (зависимость уже включена).

## Тесты

```bash
pytest
```

Тесты покрывают валидаторы ввода и структуры браузерного клиента. Они не
требуют реального бота, БД или браузера.

## Что нужно доделать для реальной работы

Браузерный клиент оставлен каркасом. Чтобы он заработал с реальным сайтом:

1. Заполнить актуальные селекторы в `browser/selectors.py` (через DevTools
   на конкретном центре/стране).
2. Реализовать методы `VFSClient._open_booking_page`, `_scan_calendar`,
   `_select_slot`, `_fill_applicant_form`, `_confirm_booking`.
3. При необходимости — подключить реальный решатель капчи в
   `browser/captcha_solver.py` (с учётом правил сервиса).
4. Настроить сохранение/восстановление сессии (модель `Session`), чтобы
   не проходить авторизацию каждый раз.
