## 🧭 Обзор

VisaBooking помогает следить за появлением свободных слотов записи в визовом
центре и оформлять заявки. Пользователь создаёт «заявку» в личном кабинете на
сайте: выбирает визовый центр, категорию и подкатегорию визы, желаемый диапазон
дат и данные одного или нескольких заявителей. Фоновый воркер периодически
проверяет доступность и сообщает о найденных слотах.

Проект построен по принципу **«бэкенд — ядро, остальные — клиенты»**: единое
ядро на Go хранит данные и API, а тяжёлая браузерная часть вынесена в отдельный
Python-воркер.

## 🏗️ Архитектура

```
                ┌─────────────┐
                │  Frontend   │  JS SPA (личный кабинет)
                │  (браузер)  │
                └──────┬──────┘
                       │ HTTP /api/*
                ┌──────▼──────┐        ┌──────────────┐
                │   Go API    │◄──────►│  PostgreSQL  │
                │  + бот TG   │        └──────────────┘
                └──────┬──────┘
                       │ HTTP /jobs (контракт WORKER_API.md)
                ┌──────▼──────┐
                │ Python      │  движок: Playwright/patchright,
                │ worker      │  выбор критериев, чтение доступности
                └─────────────┘
```

- **Frontend** — статическая SPA (HTML/CSS/JS), общается с Go API.
- **Go API + бот** — REST API, аутентификация, БД, Telegram-бот, планировщик.
- **Python-воркер** — браузерная автоматизация (изолирована в одном месте).

## 🧩 Стек

| Слой        | Технологии                                         |
|-------------|----------------------------------------------------|
| Frontend    | Чистый HTML / CSS / JavaScript (ES-модули)         |
| Backend API | Go 1.22, `net/http`, `pgx` (PostgreSQL), `bcrypt`  |
| Бот         | Go, `telebot`                                      |
| Воркер      | Python 3.11+, FastAPI, Playwright / patchright     |
| Инфра       | Docker, docker compose, Caddy (HTTPS)              |

## 🚀 Быстрый старт (разработка)

Браузер виден — удобно отлаживать. Postgres поднимается в Docker, остальное на хосте.

**Подготовка (один раз):**
```powershell
python -m venv .venv; .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
cd api; go mod tidy; cd ..
```

**Запуск (каждый раз):**
```powershell
.\dev.ps1
```
Откроется сайт на `http://127.0.0.1:8000`, воркер на `:8800`. Сервисы — в
отдельных окнах PowerShell.

## 🐳 Запуск в Docker (сервер)

```bash
docker compose up -d --build
```
Поднимутся `db`, `worker`, `api`, `bot`. Сайт — `http://<сервер>:8000`.
В контейнере браузер работает **headless**. Полный деплой с HTTPS — в `DEPLOY.md`.

## ⚙️ Конфигурация (`.env` в корне)

| Переменная           | Назначение                                   |
|----------------------|----------------------------------------------|
| `DATABASE_URL`       | Строка подключения PostgreSQL                |
| `WEBAPP_SECRET`      | Секрет для подписи cookie-сессий             |
| `BOT_TOKEN`          | Токен Telegram-бота                          |
| `VFS_BASE_URL`       | URL визового центра                          |
| `VFS_LOGIN` / `VFS_PASSWORD` | Учётные данные кабинета VFS          |
| `BROWSER_HEADLESS`   | `false` — видимый браузер (для отладки)      |
| `PROXY_SERVER`       | Прокси (опционально, против блокировок по IP)|
