# Контракт Python-воркера (для Go-бэкенда)

Воркер — отдельный HTTP-сервис вокруг движка обхода VFS. Go-бэкенд вызывает
его, чтобы проверить доступность слотов и (в будущем) забронировать.

Базовый URL по умолчанию: `http://127.0.0.1:8800`
Запуск воркера: `cd backend && uvicorn worker_app:app --port 8800`

## GET /health
Проверка живости.
```json
{ "status": "ok" }
```

## POST /jobs
Создать задание проверки доступности. Возвращает идентификатор — задание
выполняется в фоне (браузер долгий).

Запрос:
```json
{
  "login": "user@mail.ru",
  "password": "secret",
  "center": "Французский Сервисный Визовый Центр в Санкт Петербурге",
  "category": "Краткосрочная виза",
  "subcategory": "Другие краткосрочные визы",
  "date_start": "2026-07-01",
  "date_end": "2026-07-31",
  "applicants_count": 2
}
```
Ответ:
```json
{ "job_id": "9f1c…" }
```

## GET /jobs/{job_id}
Статус и результат задания.

Пока выполняется:
```json
{ "job_id": "9f1c…", "status": "running", "result": null, "message": null }
```
Успех (`status`: `done`):
```json
{
  "job_id": "9f1c…",
  "status": "done",
  "result": {
    "has_slots": true,
    "nearest": { "1": "2026-06-30", "2": "2026-07-14" },
    "matched": "2026-07-14"
  },
  "message": null
}
```
- `nearest` — ближайшая дата по числу заявителей.
- `matched` — дата, попавшая в запрошенный диапазон (или `null`).

Ошибка (`status`: `error`) — текст в `message` (напр. Cloudflare/неверный вход).

## Статусы задания
`pending` → `running` → `done` | `error`

## Рекомендации для Go-стороны
- Создал задание → опрашивай `GET /jobs/{id}` раз в 3–5 сек до `done`/`error`.
- Для периодического поиска — планировщик в Go ставит задание каждые N минут.
- В проде вместо in-memory заданий — общая очередь (Redis/NATS) и несколько
  воркеров; контракт остаётся тем же.
- Бронирование (`/jobs` с режимом записи) добавится, когда будут селекторы
  формы/календаря/оплаты — контракт расширим полем `mode` и данными заявителей.
