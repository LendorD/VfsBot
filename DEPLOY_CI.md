# CI/CD: автодеплой на сервер через GitHub Actions

При пуше в `main` GitHub заходит на твой сервер по SSH и выполняет
`git pull` + `docker compose up -d --build`. Можно запускать и вручную:
вкладка **Actions → Deploy → Run workflow**.

## 1. Новый SSH-ключ для деплоя
Старый приватный ключ восстановить нельзя — сгенерируй отдельный ключ под CI.

На своей машине (PowerShell/терминал):
```bash
ssh-keygen -t ed25519 -C "github-deploy" -f deploy_key
```
Получишь два файла: `deploy_key` (приватный) и `deploy_key.pub` (публичный).
Пароль на ключ НЕ ставь (CI не сможет его ввести) — просто Enter.

## 2. Публичный ключ — на сервер
Добавь содержимое `deploy_key.pub` в `~/.ssh/authorized_keys` нужного
пользователя на сервере. Если ещё есть доступ на сервер (по паролю/другому ключу):
```bash
ssh-copy-id -i deploy_key.pub ПОЛЬЗОВАТЕЛЬ@АДРЕС_СЕРВЕРА
# либо вручную:
cat deploy_key.pub | ssh ПОЛЬЗОВАТЕЛЬ@АДРЕС_СЕРВЕРА "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

## 3. Секреты в GitHub
Репозиторий → **Settings → Secrets and variables → Actions → New repository secret**.
Добавь:

| Секрет        | Значение                                              |
|---------------|-------------------------------------------------------|
| `SSH_HOST`    | IP или домен сервера                                  |
| `SSH_USER`    | пользователь на сервере (напр. `root` или `deploy`)   |
| `SSH_PORT`    | порт SSH (обычно `22`)                                |
| `SSH_KEY`     | **всё** содержимое файла `deploy_key` (с BEGIN/END)   |
| `PROJECT_DIR` | путь к проекту на сервере (напр. `/home/deploy/VfsBot`)|

## 4. Что должно быть на сервере (один раз)
- Установлены **git**, **Docker** и **docker compose**.
- Репозиторий склонирован в `PROJECT_DIR` и на ветке `main`.
- Создан файл **`.env`** в корне проекта (он в `.gitignore`, в репозиторий не
  попадает — заполни секреты на сервере вручную: `BOT_TOKEN`, `WEBAPP_SECRET`,
  `VFS_LOGIN`/`VFS_PASSWORD`, и т.д.).

Первый раз вручную проверь, что всё поднимается:
```bash
cd PROJECT_DIR && docker compose up -d --build
```

## 5. Дальше
Любой `git push` в `main` → деплой автоматически. Логи — во вкладке Actions.

## Заметки
- Браузерный воркер на «безголовом» сервере проходит Cloudflare хуже — смотри
  `DEPLOY.md` (вариант с Xvfb/noVNC). Сайт, API и бот деплоятся без проблем.
- Если сервер слабый, сборка тяжёлых образов (Playwright/noVNC) долгая —
  можно выделить отдельный воркер-узел.
- Безопаснее завести на сервере отдельного пользователя `deploy` (не `root`)
  и дать ему доступ к Docker.
