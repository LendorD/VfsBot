# Деплой на VPS через Docker

Сайт+API упакованы в Docker. Обратный прокси **Caddy** автоматически выдаёт
HTTPS-сертификат. БД PostgreSQL — для движка/бота (сам сайт пока на SQLite,
которая лежит в постоянном томе).

## 1. Что нужно
- VPS с Linux (Ubuntu 22.04+), доступ по SSH.
- Домен, A-запись которого указывает на IP сервера.
- Установленные **Docker** и **docker compose**.

Установка Docker (Ubuntu):
```bash
curl -fsSL https://get.docker.com | sh
```

## 2. Доставить проект на сервер
```bash
git clone <ваш-репозиторий> visabooking && cd visabooking
# или scp/rsync папкой
```

## 3. Настроить окружение
Создайте `.env` в корне (рядом с docker-compose.yml):
```env
WEBAPP_SECRET=сгенерируйте_длинную_случайную_строку
POSTGRES_PASSWORD=надёжный_пароль_БД
# для бота/движка, когда подключим:
BOT_TOKEN=...
DATABASE_URL=postgresql+asyncpg://vfs_user:надёжный_пароль_БД@db:5432/vfs_bot
```
Впишите домен в `Caddyfile` вместо `visa.example.com`.

## 4. Запуск
```bash
docker compose up -d --build
docker compose logs -f web      # посмотреть логи
```
Откройте `https://ваш-домен` — сайт работает по HTTPS.

## 5. Обновление
```bash
git pull
docker compose up -d --build
```

## 6. Важно про автоматизацию VFS на сервере
Прохождение Cloudflare лучше всего работает в **видимом** реальном Chrome.
На «безголовом» сервере это сложнее: понадобится виртуальный дисплей (Xvfb)
или отдельный воркер, и стабильность ниже, чем на твоей машине с VPN.
Рекомендуется: сайт/API и БД — на VPS, а **браузерный поиск слотов** запускать
там же через Xvfb ИЛИ держать отдельным воркером (можно даже на машине с
реальным IP/VPN). Это отдельная задача — спланируем, когда подключим движок.
